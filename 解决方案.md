# Vite Rolldown 绑定问题解决方案

## 问题描述
新版本的 Vite (8.x) 使用了 Rolldown，但其在 macOS ARM64 架构上的原生绑定可能无法正确安装。

## 解决方案

### 方案 1：手动清理并重新安装（推荐）

在你的终端中执行以下命令：

```bash
cd /Users/lovewcc/Documents/Me/Vue/vue3-naiveui

# 清理依赖
rm -rf node_modules package-lock.json

# 重新安装
npm install

# 运行项目
npm run dev
```

如果上述方法不行，尝试方案 2。

### 方案 2：使用稳定版本的 Vite

编辑 `package.json` 文件，将 Vite 和相关插件降级到稳定版本：

```json
"devDependencies": {
  "@vitejs/plugin-vue": "^5.0.0",
  "vite": "^5.4.0"
}
```

然后重新安装：

```bash
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### 方案 3：使用 pnpm 代替 npm

```bash
# 安装 pnpm
npm install -g pnpm

# 清理并重新安装
rm -rf node_modules package-lock.json
pnpm install

# 运行项目
pnpm dev
```
